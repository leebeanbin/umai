## Design Context

### Users
두 가지 사용자 계층:
1. **개인 사용자** — AI 어시스턴트로 일상적인 작업, 문서 작성, 정보 탐색에 활용. 복잡한 설정 없이 바로 사용 가능하길 원함.
2. **엔지니어 / 개발자** — 업무용으로 AI Agent, RAG, 워크플로우 자동화, 파인튜닝까지 활용. 커스터마이징 깊이와 최신 모델 지원이 중요.

사용 맥락: 주로 데스크탑, 집중 작업 세션. 저녁이나 밤에 어두운 환경에서 장시간 사용.

### Brand Personality
**3 words**: 조용하고 강력하고 신뢰감 있는

- **조용함 (Quiet)**: UI가 작업을 방해하지 않는다. 정보는 필요할 때 나타나고, 불필요한 것은 숨어있다.
- **강력함 (Powerful)**: 겉은 깔끔하지만 아래엔 Agent loop, pgvector, 파인튜닝까지 있다. 사용자가 더 파고들수록 더 많은 것이 보인다.
- **신뢰감 (Trustworthy)**: 오류 메시지가 명확하고, 상태가 항상 보이며, 내 데이터가 어디 있는지 명확하다.

감정 목표: 사용할 때 "이 도구는 내 편이다"는 느낌. 긴장이 아닌 집중.

### Aesthetic Direction
**레퍼런스**: GPT (대화 중심, 여백 충분, 기능이 숨어있음) + Notion (깔끔한 계층 구조, 컨텐츠 우선) + Linear (정밀한 다크 UI, 조용한 인터랙션)

**앤티 레퍼런스**: 
- 너무 무거운 대시보드 (너무 많은 카드, 차트, 숫자 — Grafana 느낌)
- "AI 스타트업 색감" (cyan-on-dark, 보라색 그라디언트 글로우)
- 너무 미니멀해서 차갑게 느껴지는 것 (Vercel 대시보드처럼 거리감 있는 흰 배경)

**테마**: 다크 모드 기본. 엔지니어가 밤 11시에 장시간 집중하는 환경.

**폰트 방향**: Inter 교체 필요. "조용하고 신뢰감 있는 전문성" → geometric sans-serif 계열이되 Inter/DM Sans 등 과포화 폰트 제외. Geist, Satoshi, Manrope, Neue Montreal 방향.

### Design Principles

1. **컨텐츠 우선**: UI 크롬은 작업을 방해하지 않는다. 대화, 워크플로우 캔버스, 문서가 주인공.
2. **점진적 공개**: 기본 상태는 깔끔하고 단순. 고급 기능은 인터랙션을 통해 드러난다. (설정, 파라미터, 도구 등)
3. **상태 투명성**: 생성 중, 저장 중, 에러, 완료 — 모든 상태가 명확하게 보인다. 사용자가 뭔가 일어나고 있음을 알아야 한다.
4. **조용한 인터랙션**: hover, focus, transition 등 모든 인터랙션은 자연스럽게 감속하며, 주목을 끌지 않는다.
5. **밀도 조절**: 채팅은 여백이 넉넉하고, 어드민/워크스페이스는 밀도가 높아도 괜찮다. 컨텍스트에 맞게.
